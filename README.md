# Мой первый сайт — приветственный лендинг

Небольшой статический сайт-визитка **Натальи Малахова**: канал «МИР СТИКЕРОВ И НЕЙРО», услуги, примеры работ, музыкальный проект MalSu и заявка на сотрудничество.

## Структура репозитория

| Путь | Назначение |
|------|------------|
| `index.html` | Разметка страницы (навигация, секции, форма, лайтбокс) |
| `styles.css` | Стили и адаптивная вёрстка |
| `avatar/` | Аватар в шапке |
| `image/` | Изображения для блока «Визуал и дизайн» |
| `video/` | Ролики для блока «Видео и анимация» |
| `audio/` | Аудио-фрагменты в блоке «Музыка и песни» |
| `malsu/` | Обложка и материалы альбома MalSu |

## Функционал лендинга

- Закреплённое меню с переходом по якорям по странице
- Ссылки на каналы **Telegram** и **MAX**, личные контакты
- Портфолио в виде мозаики: изображения и встроенное видео без перехода на другие сайты
- **Лайтбокс** для изображений в «Визуал и дизайн» (клик / клавиатура, закрытие по фону и Escape)
- Встроенные аудио-плееры с названиями треков
- Блок альбома MalSu с обложкой и ссылками на прослушивание
- Форма заявки с отправкой текста в **Telegram** (открывается чат с заранее заполненным сообщением)

## Как открыть локально

1. Откройте файл `index.html` в браузере (двойной клик).

Для корректной работы некоторых браузеров со встроенным аудио/видео при открытии с диска можно поднять простой HTTP-сервер в каталоге проекта (если установлен Python):

```bash
python -m http.server 8080
```

Затем в браузере: `http://localhost:8080`.

## Публикация на GitHub Pages

Сайт — чистая статика (`index.html` в корне), сборка не нужна.

1. Создайте репозиторий на [GitHub](https://github.com/new) (можно с README или пустой).
2. Установите [Git](https://git-scm.com/download/win), откройте терминал в папке проекта и выполните (подставьте свой URL репозитория):

```bash
git init
git add .
git commit -m "Лендинг: первый коммит"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
git push -u origin main
```

3. На GitHub откройте репозиторий → **Settings** → **Pages** (в боковом меню).
4. В блоке **Build and deployment** выберите **Source**: **Deploy from a branch**.
5. **Branch**: `main`, папка **`/ (root)`** → **Save**.

Через 1–3 минуты сайт будет доступен по адресу вида `https://ВАШ_ЛОГИН.github.io/ВАШ_РЕПОЗИТОРИЙ/`. Страница **Actions** покажет статус последнего деплоя, если включены GitHub Actions для Pages.

**Замечание:** если репозиторий приватный, Pages на бесплатном аккаунте может быть недоступен — тогда используйте публичный репозиторий или другой хостинг (Netlify, Cloudflare Pages и т.д.).
